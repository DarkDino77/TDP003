Project in object oriented design, by making a game

Introduction

This projekt focused on teaching object oriented design though the process of game creation. The game was made in c++ using the sfml library to abstract the complicated procces of making graphics and characters move.

The game

The game that was created is a top-down endless surviror shooter, where the player figths against an ever increasing amount of enemies that come in waves. The point of the game is to survie as long as possible against. As the player kills the enemies the player is revorded with increasing power full weapons and uppgrades.

Leard experienceses

It was noticed druing the games creation that a larger part of the code then previusly thought, could be broken down into smaller smaller classes. While some parts could be removed althogether.
  2. UML: A UML digram of the game was drawn before and revised as a better structre became apperent. The drawing of the uml diagram and its relevant components was a new and intresting experience.

  4. Smart pointers: When making the game shared_ptr where used insted of raw pointers this is to simplify memory management. However it was druing the creation of this game that we learnd that to properly use them they have there own way of making new pointers that needs to be applied.
  5. Proper return and input values: One of the challages incuterd druing the process of making the game was the preformace impact of not returning the or taking in complex datatypes as refrenses where aproprite.
